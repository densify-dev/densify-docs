You can connect to your AWS account with the following method:


<ul>
  <li>
    CloudFormation Template--You need your account credentials and an external ID, which corresponds to a password.  See <a href="/docs/WebHelp_Densify_Cloud/Content/Data_Collection_for_Public_Cloud_Systems/AWS_Data_Collection_Using_a_CloudFormation_Template">AWS Data Collection Using a CloudFormation Template</a> for details on creating the connection. Once the connections are created, you will see them in the <a href="/docs-kubex/Content/Cloudex/Cloud_Connections">Cloud Connections</a>
    page.
  </li>
  <li>
    IAM Role--An IAM role establishes a trust relationship between AWS accounts and <span class="mc-variable General.ProductName variable">Densify</span>. You need to complete the prerequisite setup in your AWS account, including the external ID and Role ARN. See <a href="/docs/WebHelp_Densify_Cloud/Content/Data_Collection_for_Public_Cloud_Systems/AWS_Data_Collection_Prerequisites_for_an_IAM_Role__CloudWatch_only_">AWS Data Collection Using IAM Roles Prerequisites</a>.
  </li>
</ul>



## Configuring a Connection to AWS



<ol start="1">
  <li>
  Click the <strong>Add</strong> button and select AWS.
  <Frame caption="Figure: Accessing the Add Button">
    <img src='/images/docs-kubex/Content/Cloudex/03000032_580x351.png'/>
  </Frame>
  </li>
  <li>
    The Densify Account ID is predefined. This value must match the value you entered when you created the role in AWS.
    <Frame caption="Figure: AWS Connection Window">
      <img src='/images/docs-kubex/Content/Cloudex/03000033_535x307.png'/>
    </Frame>
  </li>
  <li>
  Enter the AWS-specific connection parameters, listed in the table below.
  </li>
  <div class="MCDropDown MCDropDown_Closed dropDown dropDownTable">

  <Accordion title="Table: AWS Connection Parameters">
<table style={{ width: "100%", borderCollapse: "collapse" }}>
  <colgroup>
    <col style={{ width: "30%" }} />
    <col style={{ width: "70%" }} />
  </colgroup>
  <thead>
    <tr>
      <th style={{ verticalAlign: "middle" }}>Field</th>
      <th style={{ verticalAlign: "middle" }}>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td style={{ verticalAlign: "middle" }}>External ID</td>
      <td style={{ verticalAlign: "middle" }}>
        The external ID specified for Densify, when you created the IAM role in AWS.
        <br />
        If you need to edit or review a saved connection, for security reasons, you will need to re-enter the external ID.
      </td>
    </tr>
    <tr>
      <td style={{ verticalAlign: "middle" }}>Role ARN</td>
      <td style={{ verticalAlign: "middle" }}>
        The Amazon Resource Name (ARN) for the IAM role that you created in AWS.
      </td>
    </tr>
  </tbody>
</table>
  </Accordion>


  </div>
  <Note>
    <strong>Note:</strong> The External ID and the Role ARN are encrypted and are not displayed as plain text in log files.
  </Note>
  <li>
    Verify your connection by clicking <strong>Verify Connection</strong>.
    <ul>
    <li class="ListBullet">
    If the credentials are valid, you will be connected and authenticated.
    </li>
    <li class="ListBullet">
    If the credentials cannot be validated, then review the displayed error message and correct your credentials. It is possible that the user account does not have the required permissions or that the IAM role has not been setup properly. See <a href="/docs/WebHelp_Densify_Cloud/Content/Data_Collection_Troubleshooting/AWS_Cloud_Connections">Troubleshooting AWS Cloud Connections</a> for details.
    </li>
    </ul>
  </li>
  <li>
    Once the account is verified, discovers the AWS account ID and displays the ID number in a second dialog box.

  <Note>
    <strong>Note:</strong> The number of instances returned is an estimate obtained when Densify verifies the connectivity. It may not match the number of instances discovered during the more detailed, scheduled data collection.
  </Note>
  </li>

  <li>
    Optionally, specify a connection name to clearly identify the AWS account in Densify. This name will appear in the <strong>Cloud Connections</strong> list so you can easily identify this connection.
    <Frame caption="Figure: Connection Confirmation Window">
      <img src='/images/docs-kubex/Content/Cloudex/03000035_530x283.png' />
    </Frame>
    <p>The connection name must be unique within the AWS connection type section, so if the name is already in use, you are prompted to enter a new connection name.</p>
    <p>The Connection Name is limited to 32-characters.</p>
  </li>

  <li>
  Click the <strong>Save Connection</strong> button, that appears when the connection is successfully verified.
  <p>The connection table is updated with your new connection listed. Actual data collection will begin at the scheduled time, usually overnight.</p>
  </li>

  <li>If you want to add another connection, click <strong>Add</strong> and repeat these steps to configure the connection.</li>
</ol>
