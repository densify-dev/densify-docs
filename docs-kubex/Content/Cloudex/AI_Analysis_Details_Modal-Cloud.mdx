## CPU | Memory Request and Limits

These blocks lists the current and recommended allocation settings, based on utilization for the selected container. The values are colour-coded to easily identify savings and wasted resources. A dash (-) indicates no value has been set.

These values are per container if there is more than 1 container defined by the selected manifest

## Total Surplus Resources

These blocks list the aggregated CPU and memory for the selected manifest. The value is colour-coded to easily identify savings. Wasted spend is listed in the last block. A negative value indicates you need to increase the CPU or memory setting to eliminate risk.
