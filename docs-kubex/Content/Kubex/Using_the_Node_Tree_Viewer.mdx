The node tree viewer offers an intuitive and efficient way to navigate and explore your node groups. Nodes and Node Groups Node that have been discovered in the last day are displayed in the tree viewer,


<Frame caption="Figure: Locating the Node Tree Viewer">
![](/images/docs-kubex/Content/Kubex/03000129_705x348.png)
</Frame>



<InlineImageWithText href="#" src="/images/docs-kubex/Content/Kubex/03000004.png">
Use the <a href="/docs-kubex/Content/Kubex/Using_the_Tree_Viewer">container tree viewer</a> to navigate your container environment.
</InlineImageWithText>


Use the tree viewer to select the scope of nodes on which to report. As you navigate through the tree, the reports on the right update automatically to show the selected scope.

### Views and Filters

The default view is your container hierarchy i.e. Clusters > Node Groups > Individual Nodes. These views can be customized to better manage your data. See [Creating Views from a Table](#creating-from-table).

</Frame>

## Using Tabs to View the Selected Systems

Based on your selection in the tree viewer various tabs are displayed to view the selected data. When multiple systems are selected you see:


<ul>
  <li>
    <a href="/docs-kubex/Content/Kubex/Node_Group_Analysis">
    Node Groups
    </a>
    --Shows you the selected scope in a tabular view. Three views are provided and you can create customized views by selecting only the required columns and saving the results
  </li>
  <li>
    <a href="/docs-kubex/Content/Kubex/Nodes_Tab">
    Nodes
    </a>
    --Shows you the details of individual nodes. Again, the content of the report is based on your selection in the tree viewer.
  </li>
</ul>


When a single node is selected you see:


<ul>
  <li>
    <a href="/docs-kubex/Content/Kubex/Overview_Tab">
    Overview
    </a>
    --Shows you the utilization and recommendation details for a single node.
  </li>
  <li>
    <a href="/docs-kubex/Content/Kubex/GPU_Tab">
    GPU
    </a>
    --Shows you the GPU resource allocation and utilization details for a single node with configured GPU resources.
  </li>
</ul>



## Creating Views from a Table

You can also create a view directly from the table, using the columns to define your view: 


<ol start="1">
  <li>Select the columns to be added and/or removed.</li>
  <li>Drag any column to the header bar to group the data by the selected item.</li>
  <li>Once the data is organized to meet your requirements, open the context menu.</li>
  <li>Click **+Save as new view** and enter a name for your view.</li>
</ol>
